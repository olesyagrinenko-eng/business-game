# Бизнес-игра — веб-версия

Офлайн-мероприятие с партнёрами-франчайзи: 5 раундов, команды по 5–6 человек. В каждом раунде пары команд (Команда 1 и Команда 2) выбирают коэффициент изменения ставки курьера; результат зависит от решений обеих сторон.

## Запуск

1. Собрать данные из Excel (если меняли `svod_full.txt`):
   ```bash
   cd "бизнес игра/game-web"
   python3 build_data.py
   ```

2. Установить зависимости и запустить сервер:
   ```bash
   pip install -r requirements.txt
   python app.py
   ```

3. Открыть в браузере:
   - **Участники:** http://localhost:5001/ — регистрация по названию команды, затем выбор коэффициента в каждом раунде.
   - **Экран (проектор):** http://localhost:5001/screen — эту страницу открывать на большом экране (вводные, задание, переключение раунда, таблица ответов, карта из 4 квадрантов). На главной странице есть ссылка «Для экрана (проектор)».

## Механика

- Регистрация: команды вводят название. Нечётные становятся «Команда 1» в паре, чётные — «Команда 2». Количество команд должно быть чётным (пары: 1–2, 3–4, …).
- В каждом раунде на экране показываются общие и доп. вводные. Обе команды в паре выбирают коэффициент: **-10%**, **0%**, **+20%**, **+40%**.
- После того как оба ответа получены, результат (CTE, DC, в таргете ли CTE) берётся из таблицы сценариев (лист «СВОД» из Excel) и показывается командам.
- После 5 раундов на экране доступна **карта квадрантов**: по оси X — достижение таргета по CTE (больше половины раундов), по оси Y — суммарный DC (выше/ниже медианы). Четыре квадранта: высокий DC + в таргете; высокий DC + вне таргета; низкий DC + в таргете; низкий DC + вне таргета.

## Файлы

- `app.py` — Flask-сервер (регистрация, ответы, смена раунда, результаты).
- `data/scenarios.json` — общие вводные, доп. вводные по раундам и матрица сценариев (генерируется из `build_data.py` при наличии `svod_full.txt` в родительской папке).
- `static/index.html` — страница для участников (название команды, вводные, кнопки выбора коэффициента, результат раунда).
- `static/screen.html` — экран для зала (вводные, переключение раунда, таблица пар и ответов, карта квадрантов).
- `static/derevya/` — **деревья метрик**: отдельная HTML-страница на каждый раунд (1–6). На странице можно выбрать решения команд (коэффициенты); дерево автоматически подставляет значения метрик из СВОД и подсвечивает их (зелёный / жёлтый / красный). Ссылка «Деревья метрик» есть на экране для проектора. Данные для расчётов соответствуют листу СВОД (раунды 1–5 из `scenarios.json`; раунд 6 использует данные раунда 5).
- `build_data.py` — сборка `scenarios.json` из текстового дампа листа СВОД (без Excel).
- `export_scenarios.py` — альтернативный экспорт напрямую из Excel (требует openpyxl и файл `Страт.сессия_бизнес кейс_03.03.26.xlsx` в папке «бизнес игра»).

## Сеть

Для доступа с телефонов в одной сети запустите сервер (он слушает на `0.0.0.0:5001`) и откройте на устройствах адрес вида `http://<IP_ноутбука>:5001`. На macOS порт 5000 часто занят AirPlay — поэтому используется 5001.

## Деплой на Render (бесплатно)

Чтобы выложить игру в интернет на [Render](https://render.com):

1. **Репозиторий на GitHub**  
   Создайте репозиторий и залейте в него **содержимое папки `game-web`** так, чтобы в корне репо лежали `app.py`, `requirements.txt`, папки `data/` и `static/`, файл `render.yaml`.  
   Удобный вариант: создать новый репо, скопировать в него только файлы из `game-web` (без родительской папки «бизнес игра»).

2. **Подключение к Render**  
   Зайдите на [dashboard.render.com](https://dashboard.render.com) → **New** → **Web Service**. Выберите ваш репозиторий.

3. **Настройки сервиса**  
   - **Name:** например, `business-game`.  
   - **Root Directory:** оставьте пустым, если в корне репо уже лежит `app.py`; если в репо лежит вся папка проекта, укажите путь к папке с `app.py` (например, `бизнес игра/game-web`).  
   - **Build Command:** `pip install -r requirements.txt`  
   - **Start Command:** `gunicorn --bind 0.0.0.0:$PORT app:app`  
   Если в репо есть `render.yaml`, Render может подхватить эти команды сам.

4. **Create Web Service**  
   После деплоя Render выдаст URL вида `https://business-game.onrender.com`. Его можно раздавать участникам и открывать экран.

**Важно:** на бесплатном тарифе сервис «засыпает» после ~15 минут без запросов; при первом заходе после сна ответ может прийти с задержкой 30–60 секунд. Все данные (команды, ответы) хранятся в памяти и обнуляются при перезапуске или после сна.
